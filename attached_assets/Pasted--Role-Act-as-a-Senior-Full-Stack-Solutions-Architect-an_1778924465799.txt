# Role
Act as a Senior Full-Stack Solutions Architect and Lead Developer. Your task is to scaffold and build the "EdTech Study Platform" based strictly on the provided Software Requirements Specification (SRS) v7.3.

# Project Context
We are building a Progressive Web App (PWA) for competitive exam preparation (JEE, NEET, GATE). 
**CRITICAL CONSTRAINT:** We are using **externally managed infrastructure** with **OUR OWN credentials**. 
- **NO** AI-platform built-in backends or storage.
- **NO** vendor-locked storage.
- **MUST** use Supabase (Auth, DB, Edge Functions) and Backblaze B2 (Storage) with user-provided keys.

# Tech Stack
- **Frontend:** React 18, Vite, TypeScript, Tailwind CSS, Framer Motion, Zustand (State), Dexie.js (IndexedDB).
- **Backend:** Supabase (PostgreSQL, Auth, RLS, Edge Functions - Deno Runtime).
- **Storage:** Backblaze B2 (PDFs & Profile Photos) + Cloudflare CDN (Caching).
- **Email:** Resend (Transactional Emails).
- **Icons:** Lucide-React or Heroicons.
- **Forms:** React Hook Form + Zod.

# Core Infrastructure Requirements (MUST IMPLEMENT)
1. **Environment Variables:**
   - `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY` (Server-side only)
   - `B2_ACCOUNT_ID`, `B2_APPLICATION_KEY_ID`, `B2_APPLICATION_KEY`, `B2_BUCKET_NAME`
   - `RESEND_API_KEY`

2. **Authentication Flow:**
   - Registration: Full Name, **Indian Mobile (+91)**, Email, Password, Confirm Password.
   - **Validation:** Mobile must match regex `^(\+91)[\s-]?[6-9]\d{9}$`. Password must be strong (8+ chars, mixed case, number, symbol).
   - **Verification:** Email verification is MANDATORY. Users cannot log in until `email_verified = true`.
   - **Profile:** Users can upload/change/remove profile photos (stored in B2, compressed client-side).
   - **Password Change:** Requires current password verification.

3. **Storage Strategy (Backblaze B2):**
   - **PDFs:** Gated upload (only after Chapter Test completion). Path: `users/{uid}/chapters/{chapter_id}/notes_{timestamp}.pdf`.
   - **Photos:** Path: `users/{uid}/profile/photo.jpg`.
   - **Security:** All uploads/downloads via **Signed URLs** generated by Supabase Edge Functions. Never expose B2 keys to the client.
   - **CDN:** Route B2 downloads through Cloudflare CDN to reduce egress costs.

# Functional Modules to Build

## 1. Smart Gating System
- Hierarchy: Subject -> Chapter -> Topic.
- Logic: No access to DPP/PYQ/Tests until previous steps are passed.
- Enforcement: Frontend UI locks + **Server-side Edge Function validation** (return 403 if gate not passed).

## 2. Exam Engine
- Features: Timer (server-synced), Question Grid, Mark for Review, Pause/Resume, Auto-Submit.
- Offline: Save answers to IndexedDB (Dexie.js) and sync on reconnect.
- Anti-Cheating: Tab-switch warnings, server-side timer validation.

## 3. PDF & Note System
- Upload: Gated by Chapter completion.
- Search: Client-side text extraction (pdf.js) + Supabase `tsvector` indexing.
- Deduplication: SHA-256 hash check before upload.

## 4. User Profile & Dashboard
- Dashboard: Progress tree, Today's Plan, Pomodoro timer, Streaks.
- Profile Menu: View/Edit Name, Mobile, Email (read-only), Change Photo, Change Password.
- Visuals: Dark Mode First (Deep Slate `#0F172A`), Focus Indigo `#6366F1`.

# Implementation Steps

1. Initialize Vite + React + TS project.

3. Create `.env` template with all required variables.
4. Set up Tailwind config with the specific color palette from SRS (Deep Slate, Focus Indigo, etc.).

## Phase 2: Database & Auth (Supabase)
1. Write SQL migration scripts for:
   - `profiles` (extending auth.users with mobile, email_verified, full_name).
   - `subjects`, `chapters`, `topics`.
   - `user_progress` tables.
   - `user_notes` (with B2 path fields).
   - `user_profile_photos`.
   - `system_config`.
2. Implement Row Level Security (RLS) policies for all tables.
3. Create Edge Function `on-user-created` to handle Super Admin bootstrap and profile creation.
4. Create Edge Function `send-verification-email` using Resend.

## Phase 3: Storage Integration (Backblaze B2)
1. Create Edge Function `b2-upload-url`:
   - Validates user quota (500MB/user, 9GB global).
   - Validates Chapter completion (Gate Check).
   - Generates Signed Upload URL for B2.
2. Create Edge Function `b2-download-url`: Generates Signed Download URL.
3. Create Edge Function `b2-profile-photo`: Handles image compression logic and upload.
